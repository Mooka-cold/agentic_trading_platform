# 项目需求文档

## 1. 目标与范围

本项目是一个面向加密市场的 AI 交易平台，核心目标是把“信息处理”升级为“策略决策与执行闭环”。

### 1.1 目标

1. 让用户在一个界面内完成行情观察、情报解读、策略决策与执行监控。
2. 通过多智能体协作提供可解释、可追踪、可复盘的交易建议。
3. 建立可扩展的工程底座，支持新增数据源、策略模块和执行通道。

### 1.2 范围

- 包含：新闻与行情采集、AI 研判、风控审核、订单执行、日志与历史会话。
- 不包含：自定义衍生品撮合引擎、链上资产托管、策略收益保证。

## 2. 用户与场景

### 2.1 目标用户

- 个人交易者：需要更快理解市场信息，减少主观噪声。
- 半自动策略用户：需要“建议 + 风控 + 可手动接管”能力。
- 研究型用户：需要可回放日志、可解释理由、可比对策略结果。

### 2.2 核心使用场景

1. 查看实时市场状态与情绪变化，识别短期风险窗口。
2. 在多智能体辩论后获得最终建议与信心分数。
3. 在风控审核通过后执行交易，并持续监控仓位与订单。
4. 在会话结束后查看复盘结论，形成长期经验沉淀。

## 3. 功能需求

### 3.1 数据层能力

- 行情采集：支持主流交易对 K 线、深度、成交数据。
- 指标计算：支持 RSI、MACD、BB、SMA、EMA、ATR 等常见指标。
- 新闻聚合：支持 CryptoPanic 与 RSS 聚合，输出可用于分析的结构化数据。
- 数据回补：支持历史 K 线回补，保障指标连续性。
- 链上快照：支持链上指标聚合并纳入策略上下文。

### 3.2 决策层能力

- 多智能体流程：Analyst、Bull、Bear、Portfolio Manager、Reviewer、Reflector。
- 仲裁机制：冲突提案收敛为单一可执行方案或 HOLD。
- 可解释输出：每次决策必须输出理由、假设、信心。
- 记忆机制：支持经验检索与复盘写入，避免重复犯错。

### 3.3 执行层能力

- 风控门禁：校验方向、止损止盈位置、RR 下限、数据质量状态。
- 执行引擎：支持模拟交易与实盘接口抽象。
- 仓位管理：支持依据风险参数调整下单数量。
- 异常控制：支持熔断、手动停止、失败重试与状态回传。

### 3.4 交互层能力

- Dashboard：行情、新闻、智能体状态、组合表现同屏展示。
- History：按 session 回看日志、artifact、关键决策链路。
- Strategy：支持策略参数与 Agent 配置调整。
- Settings：支持系统配置、输出语言与运行参数管理。

## 4. 非功能需求

### 4.1 性能

- 前端关键页面响应应稳定在交互可接受范围内。
- 行情更新应保持秒级可见性，避免明显卡顿。

### 4.2 稳定性

- 关键服务支持失败重试与降级回退。
- 工作流异常必须可观测、可告警、可追踪。

### 4.3 安全与合规

- API Key 与敏感配置不得明文持久化。
- 日志与对外接口不得泄露用户敏感信息。
- 依赖升级必须通过安全审计与构建校验。

### 4.4 可维护性

- 模块边界清晰，支持替换单个服务而不影响整体流程。
- 文档与代码版本保持一致，更新需同步维护。

## 5. 验收标准

### 5.1 功能验收

- 能够完成一次完整工作流：采集 → 研判 → 仲裁 → 风控 → 执行/拒绝 → 复盘。
- 历史页面可按 session 查看关键日志与结构化 artifact。
- 风控拒绝时，能给出明确 reject_code 与修复建议。

### 5.2 工程验收

- 前端通过 `npm run lint` 与 `npm run build`。
- 前端依赖审计通过 `npm audit --omit=dev`。
- 核心文档（README、需求、技术、架构）术语一致。
