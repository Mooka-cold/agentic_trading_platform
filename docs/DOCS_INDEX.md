# 文档总览

本页用于快速定位文档，建议从上到下阅读。

## 1. 入门与范围

- [需求文档](./REQUIREMENTS.md)：功能边界、非功能指标、验收标准
- [产品设计](./PRODUCT_DESIGN.md)：用户角色、核心流程、页面能力映射

## 2. 设计与实现

- [技术设计](./TECHNICAL_DESIGN.md)：模块职责、接口边界、数据流与工程约束
- [系统架构 V2](./SYSTEM_ARCHITECTURE_V2.md)：面向扩展与高并发的部署架构
- [UI 规范](./UI_SPEC.md)：界面结构、组件行为、交互一致性
- [运维与发布手册](./OPERATIONS_RUNBOOK.md)：发布检查、故障排查、回滚流程

## 3. 建议阅读顺序

1. 需求文档
2. 产品设计
3. 技术设计
4. 系统架构 V2
5. UI 规范
6. 运维与发布手册
7. 链上钱包情报接入规范

## 4. 维护规则

- 涉及业务流程变更时，优先更新需求文档与产品设计
- 涉及服务边界、依赖、部署策略变更时，更新技术设计与系统架构文档
- 涉及页面结构、组件交互、视觉语义变更时，更新 UI 规范
- 涉及发布流程、依赖审计、回滚步骤变更时，更新运维与发布手册
- 涉及链上数据源、地址标签、资金流规则变更时，更新链上钱包情报接入规范
- 所有文档更新应与当前代码行为保持一致
