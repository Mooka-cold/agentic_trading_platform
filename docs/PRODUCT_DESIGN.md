# 产品设计文档

## 1. 产品定位

AI Trading Platform 是一个“决策可解释、执行可追踪、复盘可回放”的交易工作台。  
产品核心不是单次信号准确率，而是让用户持续理解市场、降低错误决策频率。

## 2. 角色与价值

### 2.1 目标角色

- 交易执行者：关注实时建议与风险提示，追求操作效率。
- 策略研究者：关注日志、artifact、历史会话，追求可复盘与可迭代。
- 运营维护者：关注系统状态、依赖安全与发布稳定性。

### 2.2 关键价值

- 把分散的数据源统一为结构化上下文。
- 把“黑盒建议”变为“有理由、有假设、有风控门禁”的输出。
- 把一次会话沉淀为可复盘资产，而不是即时聊天记录。

## 3. 核心体验流

### 3.1 交易会话主流程

1. 用户选择标的并触发会话。
2. 数据层完成行情、新闻、链上与宏观上下文准备。
3. 多智能体完成研判与对抗提案。
4. PM 输出最终提案，Reviewer 执行风控校验。
5. 通过则执行，不通过则给出修订建议并进入修订轮。
6. Reflector 输出复盘摘要与经验项。

### 3.2 用户可见反馈

- 过程日志：展示当前智能体思考阶段与状态。
- 结构化结果：展示 action、SL/TP、置信度、拒绝码、修复建议。
- 历史回放：按 session 回看完整链路与关键 artifact。

## 4. 页面设计

### 4.1 Dashboard

- 目标：在单屏内完成“看盘 + 看因子 + 看智能体 + 看组合”。
- 核心区块：
  - 市场总览与关键指标
  - 智能体工作区（含策略委员会聚合视图）
  - 组合与仓位侧栏
  - 新闻与情绪摘要

### 4.2 Strategy

- 目标：统一管理策略参数与 Agent 侧行为配置。
- 核心能力：
  - 配置编辑与保存
  - 钱包连接与身份状态
  - 参数变化的可追踪性

### 4.3 History

- 目标：快速定位某一会话为何通过、为何拒绝、为何执行失败。
- 核心能力：
  - 条件筛选（symbol、status、review_status）
  - 会话详情（日志、artifact、阶段输出）
  - 决策链路可解释展示

### 4.4 Settings

- 目标：统一管理系统级开关、接口配置与运行策略。
- 核心能力：
  - 工作流运行参数
  - 输出语言与系统配置
  - 依赖与运行基线提示

## 5. 设计原则

### 5.1 可解释优先

- 每个关键决策都需要有文本理由和结构化字段。
- 风控拒绝必须带 reject_code 与 fix_suggestions。

### 5.2 渐进展示

- 首屏优先状态和结论，详情按需展开。
- 复杂 artifact 以可读摘要 + 原始 JSON 双层呈现。

### 5.3 稳定反馈

- 成功、警告、失败状态使用统一语义和视觉表达。
- 任何执行失败都必须落日志并可在 History 回查。

## 6. 版本演进方向

- 短期：继续提升会话可解释性与修订闭环收敛效率。
- 中期：增强策略对比、风控模拟和批量回放能力。
- 长期：面向多市场与多账户管理，形成统一交易中台体验。
